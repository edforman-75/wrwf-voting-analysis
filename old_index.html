Pooled Fund Grants Voting System

Overview

This project examined how WRWF members vote to allocate pooled fund grants, with the goal of making the process more expressive and meaningful while remaining simple for 245+ members to complete.

The research explored:

  • Limitations of the current ranked-choice voting system (rank 1-15)
  • Alternative approaches used by community foundations, municipalities, and member organizations
  • How to ensure equal voice regardless of contribution level
  • Ballot secrecy and voter authentication

What's available here:

  • A white paper analyzing voting approaches with references to organizations using each method
  • Interactive prototype ballots for four different voting systems
  • A process flow chart showing the complete voting workflow

The System is Built

A complete, working voting application has been developed — Flask/Python with SQLite, supporting all four voting approaches. It includes personal voting links, auto-save drafts, ballot secrecy, an admin dashboard, and CSV export. The source code is in a GitHub repository, ready for deployment. To use it for actual WRWF voting would require integration with the member email list and hosting on a WRWF domain.

Key Features of the System

Auto-Save Drafts

Voters can start their ballot, close their browser, return days later from a different device, and pick up exactly where they left off. No lost work.

Ballot Secrecy

At submission, the voter's identity is permanently severed from their ballot. Even admins cannot see how any individual voted.

Personal Voting Links

Each member receives a unique link by email. No passwords to remember — clicking the link logs them in automatically.

Real-Time Admin Dashboard

See participation stats, send reminders to non-voters, export results — all from a simple web interface.

Integration with WRWF Infrastructure

The voting system is a standalone Flask/Python application that can be hosted anywhere. Studio360, the local firm that built the WRWF website, could handle the technical integration:

  • Host on a WRWF subdomain (e.g., vote.woodriverwomensfoundation.org)
  • Connect to existing member email lists for ballot distribution
  • Style to match the main WRWF website branding
  • Set up SSL certificate for secure voting

The technical handoff is straightforward — all code is in a single GitHub repository with documentation. Alternatively, the system can be hosted on a simple cloud service (PythonAnywhere, Render, etc.) without Studio360 involvement.
