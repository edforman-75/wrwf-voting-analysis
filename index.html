Pooled Fund Grants Voting System v2

Overview

This project examined how WRWF members vote to allocate pooled fund grants, with the goal of making the process more expressive and meaningful while remaining simple for 245+ members to complete.

The research explored:

  • Limitations of the current ranked-choice voting system (rank 1-15)
  • Alternative approaches used by community foundations, municipalities, and member organizations
  • How to ensure equal voice regardless of contribution level
  • Ballot secrecy and voter authentication

What's available here:

  • A white paper analyzing voting approaches with references to organizations using each method
  • Interactive prototype ballots for four different voting systems
  • A process flow chart showing the complete voting workflow

Participatory Budgeting

Tiered Allocation

Quadratic Voting

100 voice credits, but expressing stronger preferences costs more. Prevents single-issue dominance while allowing intensity.

Current Ranked System

Rank 1-15 from the full applicant list. Rank #1 = 15 points, #15 = 1 point. Included for comparison.

Integration with WRWF Infrastructure

The voting system is a standalone Flask/Python application that can be hosted anywhere. Studio360, the local firm that built the WRWF website, could handle the technical integration:

  • Host on a WRWF subdomain (e.g., vote.woodriverwomensfoundation.org)
  • Connect to existing member email lists for ballot distribution
  • Style to match the main WRWF website branding
  • Set up SSL certificate for secure voting

The technical handoff is straightforward â€” all code is in a single GitHub repository with documentation. Alternatively, the system can be hosted on a simple cloud service (PythonAnywhere, Render, etc.) without Studio360 involvement.
